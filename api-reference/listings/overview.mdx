# Business Listings

The Listings API allows you to search business listings across multiple jurisdictions and claim them for your organization.

## The Listing Object

```json
{
  "id": "list_123abc",
  "source": "SOS",
  "entityId": "C1234567",
  "name": "Acme Corporation",
  "jurisdiction": "CA",
  "registrationDate": "2020-01-15",
  "status": "ACTIVE",
  "matchConfidence": 0.95,
  "claimStatus": "UNCLAIMED",
  "registeredAgent": {
    "name": "John Doe",
    "address": {
      "line1": "123 Main St",
      "city": "San Francisco",
      "state": "CA",
      "zip": "94105"
    }
  }
}
```

## Search Listings

<CodeGroup>
```bash cURL
curl -X GET "https://api.palm.com/v1/listings/search?query=acme&jurisdiction=CA" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```typescript Node
const listings = await palm.listings.search({
  query: 'acme',
  jurisdiction: 'CA'
});
```
</CodeGroup>

## Search Parameters

| Parameter | Description | Example |
|-----------|-------------|---------|
| `query` | Business name or ID | `acme` |
| `jurisdiction` | State code | `CA` |
| `status` | Registration status | `active` |
| `entityType` | Type of entity | `corporation` |
| `page` | Page number | `1` |
| `limit` | Results per page | `10` |

## Claim a Listing

```bash
curl -X POST "https://api.palm.com/v1/listings/{listingId}/claim" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "verificationMethod": "DOCUMENT",
    "documents": [
      {
        "type": "FORMATION",
        "file": "base64_encoded_file"
      }
    ]
  }'
```

## Claim Status

| Status | Description |
|--------|-------------|
| `UNCLAIMED` | No active claims |
| `PENDING` | Claim under review |
| `VERIFIED` | Claim approved |
| `REJECTED` | Claim rejected |
| `DISPUTED` | Claim disputed |

## Error Handling

<CodeGroup>
```json 400 Invalid Claim
{
  "success": false,
  "error": {
    "code": "INVALID_CLAIM",
    "message": "Cannot claim inactive business",
    "details": {
      "listingId": "list_123abc",
      "status": "INACTIVE"
    }
  }
}
```

```json 409 Existing Claim
{
  "success": false,
  "error": {
    "code": "EXISTING_CLAIM",
    "message": "Listing already has an active claim",
    "details": {
      "listingId": "list_123abc",
      "existingClaimId": "claim_456def"
    }
  }
}
```
</CodeGroup>

## Best Practices

1. **Search First**: Always search for existing listings before creating new profiles
2. **Verify Information**: Cross-check listing details with official records
3. **Document Ready**: Have verification documents prepared before claiming
4. **Monitor Status**: Set up webhooks to track claim status changes
