# Business Listings

The Public Listings API provides access to a comprehensive database of business entities aggregated from authoritative government sources. This system decouples public data aggregation from private business workflows, creating a robust and secure system for managing business information.

## Quick Start

### Search for Listings

<CodeGroup>
```bash cURL
curl -X GET "https://api.palm.com/v1/listings/search?query=acme&jurisdiction=CA" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```typescript Node
const listings = await palm.listings.search({
  query: 'acme',
  jurisdiction: 'CA'
});
```

```python Python
listings = palm.listings.search(
    query='acme',
    jurisdiction='CA'
)
```
</CodeGroup>

### Get a Specific Listing

<CodeGroup>
```bash cURL
curl -X GET "https://api.palm.com/v1/listings/list_123abc" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```typescript Node
const listing = await palm.listings.retrieve('list_123abc');
```

```python Python
listing = palm.listings.retrieve('list_123abc')
```
</CodeGroup>

## Core Data Models

### Business Listing Object

```typescript
interface BusinessListing {
  id: string;                    // Unique identifier
  business_name: string;         // Legal business name
  dba_names: string[];          // Doing Business As names
  entity_type: string;          // Business structure type
  formation_state: string;      // State of formation
  formation_date: string;       // Date of formation
  status: string;              // Current status
  registered_address: Address;  // Primary address
  mailing_address?: Address;    // Optional mailing address
  entity_number: string;       // State registration number
  jurisdiction: string;        // Registration jurisdiction
  registered_agent?: {
    name: string;
    address: Address;
  };
  data_sources: DataSourceMetadata[];
  industry_categories?: string[];
  location_coordinates?: {
    latitude: number;
    longitude: number;
  };
  website?: string;
  social_profiles?: SocialProfile[];
  last_verified: string;
  confidence_score: number;
  data_completeness: number;
  is_claimed: boolean;
  claim_status?: 'pending' | 'verified' | 'rejected';
  has_profile?: boolean;
}
```

### Search Parameters

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `query` | string | Business name or ID | `acme` |
| `jurisdiction` | string | State code | `CA` |
| `status` | string | Registration status | `active` |
| `entityType` | string | Type of entity | `corporation` |
| `page` | number | Page number | `1` |
| `limit` | number | Results per page | `10` |

### Initiate a Claim

<CodeGroup>
```bash cURL
curl -X POST "https://api.palm.com/v1/listings/{listingId}/claim" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "verificationMethod": "DOCUMENT",
    "documents": [
      {
        "type": "FORMATION",
        "file": "base64_encoded_file"
      }
    ]
  }'
```

```typescript Node
const claim = await palm.listings.claim(listingId, {
  verificationMethod: 'DOCUMENT',
  documents: [{
    type: 'FORMATION',
    file: documentFile
  }]
});
```
</CodeGroup>

## Best Practices

1. **Search First**: Always search for existing listings before creating new profiles
2. **Verify Information**: Cross-check listing details with official records
3. **Document Ready**: Prepare verification documents before claiming
4. **Monitor Status**: Use webhooks to track claim status changes
5. **Handle Errors**: Implement proper error handling for all API responses

## Data Quality Features

- **Normalization**: Standardized data format across sources
- **Deduplication**: Advanced entity matching prevents duplicates
- **Enrichment**: Additional data points from various sources
- **Confidence Scoring**: Quality scores based on verification status
